# Getting Started

## 1. Fork the repository

Create a public fork of `dchorazkiewicz/Math_Problems_Repo` on your own GitHub account. All of your notes and commits belong in your fork, not in the instructor's source repository.

## 2. Connect your fork to AI

If your AI tool supports a GitHub connection, authorise access to your own fork. The exact available actions depend on the tool and its permissions.

## 3. Read the course rules

Before solving problems, read:

- `NOTE_GUIDELINES.md`,
- `AI_WORKFLOW.md`,
- the selected `problem_XX.md` file.

The authoritative problem statements are kept in the [Basic Mathematics PDF](https://github.com/dchorazkiewicz/Lectures_notes_old_school/blob/main/mathematical_lectures/basic_mathematics/main.pdf), not in the empty repository templates. Copy only the statement of the problem you are beginning into its matching file.

## 4. Work systematically

Each week contains twenty problems. Work on one problem at a time, but complete the entire assigned set. Commit reviewed notes regularly rather than uploading everything shortly before class.

## 5. Present and revise

During exercise classes, be ready to display and explain any prepared solution. Feedback given to one student is useful to the whole group. Improve incomplete notes in later commits.

## Assessment

The exercise component evaluates systematic preparation, complete problem sets, mathematical justification, presentation, use of feedback, and development over time. The final examination is assessed separately and verifies independent mathematical knowledge without AI support.
